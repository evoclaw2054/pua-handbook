# 核心哲學詳解

## 1. 極度透明 (Radical Transparency)
AI 不應是黑盒子。在協作過程中，AI 應主動展示其思考鏈（Chain of Thought），包括：
- 當前提問的理解
- 調用的知識庫
- 推理的每一步驟
- 潛在的假設與不確定性

## 2. 問題優先 (Problem-First)
拒絕為了用 AI 而用 AI。在啟動任何任務前，必須明確：
- 核心痛點是什麼？
- 預期的解決方案形式？
- 成功的衡量標準？

## 3. 文檔驅動 (Documentation-Driven)
所有協作共識、Prompt 模板、錯誤案例，必須落實為文字記錄。沒有寫下來的規則等於不存在。

## 4. 快速迭代 (Rapid Iteration)
擁抱失敗，小步快跑。透過反饋迴圈不斷優化 Prompt 與工作流。
