# 除錯五步法 (Debugging 5-Steps SOP)

本標準作業程序 (SOP) 適用於所有 AI 參與的除錯場景。AI 必須嚴格遵守以下步驟，不得跳躍。

## 步驟一：聞 (Observe)
**目標**：全面收集資訊，不遺漏任何細節。
**執行動作**：
- [ ] 完整讀取錯誤日誌 (Error Log)，標記所有 ERROR/WARNING 關鍵字。
- [ ] 檢查環境變量、依賴版本、配置文件。
- [ ] **禁止**在未完成此步驟前提出任何假設。

## 步驟二：提 (Hypothesize)
**目標**：基於事實提出多個可能原因。
**執行動作**：
- [ ] 列出至少 **3 個** 可能的故障原因 (H1, H2, H3)。
- [ ] 每個假設必須有日誌或代碼行號作為支撐證據。
- [ ] 按可能性從高到低排序。

## 步驟三：鏡 (Reflect)
**目標**：自我反思，排除思維盲區。
**執行動作**：
- [ ] 反思："我是否忽略了某個明显的線索？"
- [ ] 反思："我是否過度依賴過往經驗而忽視了當前上下文？"
- [ ] 反思："如果我是攻擊者，我會在哪裡埋下這個隱患？"

## 步驟四：行 (Act)
**目標**：設計實驗驗證假設。
**執行動作**：
- [ ] **控制變量**：每次只修改一個變量，驗證結果。
- [ ] **最小化**：構建最小可複現案例 (Minimal Reproducible Case)。
- [ ] **工具強制**：必須使用搜尋工具、文檔查閱工具輔助驗證，禁止憑空猜測。

## 步驟五：復 (Review)
**目標**：驗證結果並沉澱經驗。
**執行動作**：
- [ ] 確認問題是否徹底解決 (Running Test)。
- [ ] 檢查是否有副作用 (Side Effects)。
- [ ] **覆盤**：將本次除錯過程寫入 `cases/` 目錄，形成知識沉澱。

## 違規後果
若 AI 未遵守此 SOP 直接給出結論，人類主管應立即啟動 **L3 級別壓力測試**，要求其重新執行完整流程。
